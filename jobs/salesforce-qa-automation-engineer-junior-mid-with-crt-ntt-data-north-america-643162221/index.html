We are currently seeking a Salesforce QA Automation Engineer (Junior/Mid) with CRT - Remote Canada Position We’re looking for a Salesforce QA Automation Engineer – Junior/Mid with Salesforce (SFDC) domain knowledge and a strong programming/automation background. You’ll help ensure quality across Salesforce implementations, integrations, and releases, with a strong preference for Copado Robotic Testing (CRT).

Responsibilities

  • Create and execute test plans, test cases, and test cycles for Salesforce (Sales/Service Cloud) features and configurations.
  • Build, enhance, and maintain automated regression suites using Copado Robotic Testing (CRT) and/or Selenium with Python/Java.
  • Validate Salesforce configuration changes (Flows, validation rules, security model, layouts, permissioning).
  • Test Salesforce customizations (Apex/LWC) and integrations (REST/SOAP APIs).
  • Perform defect triage, root-cause analysis, and drive closure with admins/dev teams.
  • Support UAT and communicate quality status (coverage, risks, release readiness).
  • Contribute to QA standards: automation strategy, reusable frameworks, CI/CD quality gates.

Required Qualifications

  • 2–5 years QA experience (strong junior candidates considered with relevant automation + SFDC exposure).
  • Salesforce testing experience (config + customization).
  • Automation/programming skills in Python and/or Java; experience with Selenium (or similar).
  • Agile experience and familiarity with Jira/Azure DevOps.
  • Salesforce Administrator certification (ADM-201) – required.

Preferred Skills

  • Copado Robotic Testing (CRT) experience — highly desired.
  • If no CRT experience: proven ability to scale up quickly on new automation tools and adopt a CRT-style automation mindset (modular, reusable, maintainable automation; sound test data practices).
  • Sales Cloud and/or Service Cloud experience/certifications preferred.
  • API testing (Postman, REST Assured, pytest, etc.).
  • CI/CD integration of automated tests (Jenkins/GitHub Actions/Azure DevOps).
