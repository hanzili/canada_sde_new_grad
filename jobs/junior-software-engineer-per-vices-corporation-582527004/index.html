About the Role

Develop embedded software and drivers for SDR

About Per Vices Corporation

Embedded hardware startup with Linux/CI culture

Full Description

Per Vices is looking for a software engineer who can help us build Software Defined Radios (SDR) for mission critical systems. Our customers and products use high throughput (>1GBps), low latency (<200 μ s), high performance software applications to implement various types of end applications. The applicant must be able to write clean, easy to understand sustainable code, have effective software and hardware debugging skills, regularly uses a Linux distribution, and works well in a diverse team. The applicant will apply their working knowledge in the fields of microcontroller development, continuous integration/development and embedded Linux (including device driver development).

Mandatory Qualifications

Experience in all of the following areas:

  • Experience with C or C++

Minimum Qualifications

Experience in three or more of the following areas:

  • Design, integrating, and interfacing to external chip sets, processors, and/or micro controllers in C
  • Hardware or software debugging
  • Linux kernel or device driver development experience
  • Product quality assurance testing
  • Using Yocto project
  • Using CI/CD tools (e.g. Jenkins)
  • Use of any automated testing harness or tool
  • Writing technical support documentation, or product conformity / testing reports
  • High performance computing software development experience

Optional Qualifications

The following skills are desired but not necessarily required for the position:

  • Familiarity with version control software (e.g. git)
  • Familiarity with python, matlab
  • Software optimization using profilers
  • Kernel hacking and driver development
  • RF fundamentals
  • Basic knowledge of wireless radio networks
  • Mixed signal board layout and schematic capture
  • PCB development process (from schematics, component selection, assembly, etc).
  • Multilingual
