Junior Software Quality Engineer

at Helic & Co.

📍 Location None Remote
📋 Details Full-time
📅 Posted Apr 24, 2026

About the Role

Design and execute software tests

About Helic & Co.

QA-focused software firm with growth opportunity

Full Description

Job Summary

We are seeking a detail-oriented and analytical Junior Software Quality Engineer to support software testing and quality assurance processes. In this role, you will help ensure that applications meet functional and performance requirements by executing test cases, identifying defects, and collaborating with development teams. This is an excellent opportunity for someone starting a career in software quality and testing.

Key Responsibilities

  • Design, write, and execute test cases and test plans based on requirements
  • Perform manual testing of web, mobile, or desktop applications
  • Identify, document, and track bugs using issue-tracking tools (e.g., Jira)
  • Collaborate with developers to reproduce issues and verify fixes
  • Assist in developing and maintaining automated test scripts
  • Conduct regression, functional, and performance testing
  • Participate in sprint planning, reviews, and QA activities in Agile environments
  • Ensure software meets quality standards before release
  • Maintain clear and detailed test documentation

Qualifications

  • Bachelor’s degree in Computer Science, Software Engineering, or a related field
  • 0–2 years of experience in software testing or quality assurance (internships count)
  • Basic understanding of software development lifecycle (SDLC) and QA methodologies
  • Familiarity with testing tools and frameworks
  • Strong analytical and problem-solving skills
  • Attention to detail and a commitment to quality
  • Good communication and teamwork abilities

Preferred Skills

  • Basic knowledge of programming or scripting (e.g., Java, Python, JavaScript)
  • Experience with test automation tools (e.g., Selenium, Cypress)
  • Familiarity with API testing tools (e.g., Postman)
  • Understanding of Agile/Scrum methodologies
  • Exposure to CI/CD pipelines
